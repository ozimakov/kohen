Kohen

Config from a dedicated git repo

For apps that consume domain-specific config from their own config repository — not the deployment repo. Kohen renders the path, mounts the ConfigMap, and rolls once when the version changes.

When to use it Getting started

The pattern

Keep domain and environment config in a dedicated git repo. Deploy the app with GitOps. Kohen bridges the two: path → ConfigMap → mount → stamp → roll. 

Config repo                     Deploy / GitOps
(domain + env config)           (app manifests)
        │                              │
        │ path                         │ Deployment
        ▼                              ▼
     ┌──────┐                   ┌──────────────┐
     │Kohen │── ConfigMap ─────▶│   Workload   │
     └──────┘── mount+rollout ─▶└──────────────┘

When to use it

Use Kohen when a dedicated config repo should drive a workload's ConfigMap, secret wiring, and rollouts — especially alongside Argo CD or Flux. Skip it when GitOps already owns the ConfigMap from the same repo, or when you only need secrets or restart-on-change.
