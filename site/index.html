Kohen

Git config that rolls with the workload

Point a ConfigSync at a repo path and a Deployment. Kohen renders the ConfigMap, wires the mount, and stamps exactly one rolling update when the version changes.

Install v1.0 Getting started

One job

Kohen is not another GitOps control plane. It coexists with Argo CD and Flux, owning only the config surface your app consumes — with stamps you can read and rollbacks you already know (git pins).

v1.0 artifacts

Multi-arch operator image, signed releases, Helm chart, and plain manifests (cluster + namespaced scopes). Acceptance criteria A1–A12 gate every cut.
