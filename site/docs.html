Kohen

What is Kohen

A Kubernetes operator for apps that consume domain-specific config from a dedicated git repository — separate from the deployment repo. Kohen renders a ConfigMap, mounts it, and rolls the workload when the version changes.

The main pattern

Config repo                     Deploy / GitOps
(domain + env config)           (app manifests)
        │                              │
        │ path                         │ Deployment
        ▼                              ▼
     ┌──────┐                   ┌──────────────┐
     │Kohen │── ConfigMap ─────▶│   Workload   │
     └──────┘── mount+rollout ─▶└──────────────┘

Not a GitOps control plane: deploy what runs with Argo CD / Flux; keep config in sync with Kohen.

When to use it — and when not

ScenarioUse Kohen?
Dedicated config repo drives ConfigMap + secret wiring + rolloutsYes
GitOps deploys the app; config lives in a separate config repoYes
GitOps already owns the app and its ConfigMap from the same repoNo
Config larger than ~1 MiB / large file treesNo
Secrets-only, or restart-on-change for a hand-authored ConfigMapNo
